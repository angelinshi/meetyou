## 1. 项目背景与目标

美柚作为国内头部经期健康管理 App，核心用户群为 18-35 岁女性。在用户研究中发现，大量用户希望将经期状态共享给伴侣，以便获得更贴心的关爱，同时减少因不了解对方身体状态而产生的误解与摩擦。

「经期男友版」以此为切入点，在美柚现有产品体系内，新增**男女双视角的伴侣空间**，让伴侣以男友身份加入，获取专属数据视图、发送互动关怀、共同记录恋爱生活。

> 💡 **核心价值主张：** 不改变女生原有使用习惯，以最小侵入方式，为伴侣提供「懂她、陪她」的专属空间。

### 设计目标

| 目标 | 说明 |
|---|---|
| 🔗 降低沟通成本 | 男友无需主动询问，周期状态主动呈现 |
| 💌 增强情感连接 | 互动卡片、恋爱记录沉淀两人共同记忆 |
| 🛡️ 保护女生隐私 | 数据可控，解绑后记录归零或选择保留 |
| 🎯 轻量化接入 | 男友无需深度使用 App，微信一键邀请加入 |

## 2. 核心用户场景

本次设计覆盖经期业务语境，围绕「发起关系→共同记录→关系维护→关系终止」的全生命周期展开。

### 女生视角（发起方）

- 首页查看经期状态，主动发起伴侣邀请
- 邀请成功后，首页新增恋爱记模块入口，展示最近共同记录
- 在恋爱记页面发布图文、语音，接收男友互动，共同沉淀记录
- 在「我」Tab 管理个人信息及伴侣关系

### 男生视角（接受方）

- 微信收到邀请链接，授权登录后进入男友主页
- 主页展示「今日密报」卡片：周期阶段、怀孕几率等精简数据
- 底部导航：美柚 / 日历 / 恋爱记 / 消息 / 我
- 恋爱记 Tab 查看双方记录，可发布互动（抱抱/亲亲/贴贴/想你/爱你）及图文

> 📌 男生视角核心设计原则：**最低学习成本**。男友无需了解经期专业知识，所有健康数据经过语义化处理，呈现为易懂的关心提示。

## 3. 产品架构与流程

### 整体架构（双视角）

| 层级 | 女生视角 | 男生视角 |
|---|---|---|
| **一级导航** | 推荐 / 记录 / 消息 / 我 | 美柚 / 日历 / 恋爱记 / 消息 / 我 |
| **主页** | 周期 Hero 卡 + 内容流 | 今日密报卡片 + 恋爱记预览 |
| **核心功能** | 经期记录、发布图文/语音 | 查看密报、发送互动、共同记录 |
| **关系管理** | 我 Tab → 头像 → 伴侣 → 解绑 | 我 Tab → 个人资料 → 伴侣关系 |

### 伴侣绑定流程

```
女生首页/记录Tab → 发起邀请码/二维码 → 微信分享给男友 → 男友点击链接 → 授权登录美柚 → 成功进入男友主页
```

### 邀请前 vs 邀请后差异对比

| 位置 | 未绑定状态 | 已绑定状态 |
|---|---|---|
| 女生首页 | 邀请伴侣胶囊 Banner | 恋爱记首页模块（近期记录） |
| 女生记录 Tab | 邀请引导 Banner | 正常内容流 |
| 女生顶部导航 | 搜索 + 美柚 Logo + 发帖 | 搜索 + 美柚 Logo + 发帖 |
| 男生主页 | —（不存在） | 今日密报 + 恋爱记入口 |

## 4. 功能模块详解

### 4.1 今日密报（男生主页核心卡片）

以可视化方式呈现伴侣当日经期状态，避免数字化专业词汇，使用自然语言描述：

- **周期阶段环形图**：以圆弧可视化展示女友所处经期阶段
- **怀孕几率提示**：精简语义化，如「今天很安全」
- **情绪/症状提示**：根据女生记录的症状，给出关心建议

### 4.2 恋爱记（双视角共享空间）

时间轴 Feed 形式，展示双方发布的所有记录：

| 类型 | 说明 |
|---|---|
| 📝 图文记录 | 支持文字 + 最多 9 张图片，选中序号显示 |
| 🎙️ 语音记录 | 录音后可选发送语音或 AI 一键转文字 |
| 🫂 互动卡片 | 抱抱/亲亲/贴贴/想你/爱你，随机诗意文案 |
| 💬 评论回应 | 在卡片底部回复，引用原文，可选 6 种表情点赞 |

### 4.3 互动表情体系

| 互动类型 | Emoji | 发出文案（示例） |
|---|---|---|
| 抱抱 | 🫂 | 「把你揣进口袋，就不怕失去了」 |
| 亲亲 | 💋 | 「把最甜的一颗星星摘下来，偷偷放进你口袋」 |
| 贴贴 | 🧸 | 「所有灯火里，最亮的那个是你」 |
| 想你 | 💭 | 「想你是一件很私密的事，却总被眼睛出卖」 |
| 爱你 | ❤️ | 「我把所有的运气，都用来遇见你了」 |

> ✨ 互动文案经过多轮迭代，最终确定诗意风格，每种互动类型配备 6-8 条随机文案，避免重复感。

### 4.4 点赞机制设计

- **入口**：点击卡片右下角 ❤ 图标，展开 6 个表情选择浮层（方案 B：半圆弧展开）
- **自己的卡片**：收到点赞后展示表情，点击表情触发掉落动效（非编辑）
- **对方的卡片**：点赞后不展示编辑按钮，再次点击取消点赞
- **权限**：只能点赞对方发布的内容，不能给自己点赞

### 4.5 AI 语音转文字

录音停止后，进入「预览阶段」，底部展示三个操作：**重录 / 转文字（AI处理中） / 发送语音**

转写完成后展示文字结果，用户可选「重录」或「使用文字」发布为图文帖。

## 5. 设计探索与方案比较

### 5.1 发布器入口方案（4 轮探索）

| 方案 | 交互形式 | 说明 |
|---|---|---|
| 方案 1 | 直接进入发布页 | 点击加号直接跳转全屏图文发布页，操作路径最短 |
| 方案 2 | 三键展开 | 加号垂直展开「互动/图文/语音」，适合多功能入口 |
| **方案 3 ★** | **扇形展开** | 仿参考网站动效，三按钮扇形弧出，视觉最优雅 |
| 方案 4 | 底部固定输入框 | 仿微信样式，左侧语音切换+右侧互动按钮+大文本框弹起 |

### 5.2 女生视角首页方案

| 方案 | 说明 |
|---|---|
| **方案 A ★ 毛玻璃 + 双头像** | 顶部天气背景，女生头像叠在男生头像上方，毛玻璃效果 |
| 方案 B 无天气 · 模块入口 | 不展示天气背景，已邀请后展示恋爱记首页模块（对比方案） |

### 5.3 发布器记录样式方案

| 方案 | 说明 |
|---|---|
| 方案 1 信封样式 | 卡片呈现为可翻折信封，有仪式感但交互复杂 |
| **方案 2 ★ 卡片样式** | 固化为简洁卡片，信息层级清晰，移除样式切换功能 |

### 5.4 点赞入口方案

| 方案 | 说明 |
|---|---|
| 方案 A 多表情直列 | 5 个表情直接展示在卡片上，手机端触控区偏小 |
| **方案 B ★ 半圆弧展开** | 单入口点开选择，半圆弧展开 6 表情，手机端体验更佳 |

### 5.5 男生视角恋爱记 Tab 探索

探索文件：`bf_diary_tab_explore.html`，最终方案：

- 保留顶部渐变 Hero 区，上滑后切换为不透明顶部导航栏（方案 A）
- 状态栏始终可见，滚动时顶部栏遮挡底层内容
- 移除返回按钮，作为一级 Tab 页面

## 6. 交互细节设计

### 6.1 表情雨动效

发送互动或点赞时，触发全屏表情雨：

- 对应 Emoji 从顶部随机位置落下，带旋转动画
- 穿插星星 ⭐ 和小爱心 💗 作为装饰粒子
- 粒子数量约 20-30 枚，持续约 2 秒后消失

### 6.2 删除二次确认弹窗

删除记录时弹出标准弹窗，遵循美柚设计规范：

- 主文案：「确认删除这条记录？」
- 副文案：「删除后不可恢复」
- 主按钮（红色）：确认删除；取消按钮在左侧弱化

### 6.3 解绑弹窗（女生视角）

> ⚠️ 「伴侣将无法查看你的经期等信息，恋爱记的 **N** 条记录将被清除。」
> N 为实时读取当前记录条数。

### 6.4 诗意互动文案随机机制

每种互动类型内置 6-8 条文案，发送时 Fisher-Yates 洗牌后轮换，同一文案不连续出现两次，确保多样性。

### 6.5 底部导航小红点

- 恋爱记 Tab 有未读记录时显示小红点
- 点击恋爱记 Tab 后小红点消失（清除未读状态）
- 日记 Tab 的小红点独立于点击状态，由内容更新驱动

## 8. 解绑与数据恢复机制

用户研究表明，伴侣关系的终止是一个敏感且高风险的操作。本次设计为解绑流程专门设计了完整的 14 屏流程原型（`Meetyou_Unlink_Flow.html`）。

### 解绑流程（6步）

```
S1 确认解绑 → S2 原因选择 → S3 数据处理 → S4 冷静期 → S5 最终确认 → S6 解绑完成
```

### 数据存储方案

> 💾 解绑后数据可选择保留在「感情存档」中，有效期 90 天。若双方在有效期内重新绑定，可选择**恢复所有历史记录**（包括互动、图文、语音）。

### 恢复流程（4步）

```
R1 发起恢复 → R2 选择记录 → R3 双方确认 → R4 恢复完成
```
