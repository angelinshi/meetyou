## 美柚设计规范（Meiyou Design Language for AI）

> 来源：uploads/DESIGN-meiyou-510.md + uploads/SKILL-meiyou-510.md
> 每次设计或写 UI 代码前，先按以下规范工作。

### 执行顺序
1. 判断页面类型：产品功能页、健康数据页、社区内容页、工具记录页、电商/会员页、运营活动页、弹窗/浮层、组件库页。
2. 判断业务语境：经期、备孕、怀孕、育儿、医疗健康、会员、通用平台。
3. 判断信息密度：高频工具和交易页偏紧凑；首页、健康报告、运营页可更舒展。
4. 建立 token：先定义颜色、字体、间距、圆角、阴影、组件状态，再开始布局。
5. 先做真实产品骨架：导航、返回、主内容、操作区、空/错/加载/禁用状态齐全。
6. 最后加氛围：业务色、插画、渐变、动效只能服务信息层级，不能压过内容。
7. 输出前自检：移动端 360/375/390/430 宽度无溢出、无文字重叠、触控区达标。

### 品牌气质
美柚不是"粉色皮肤的通用 App"。核心气质：**女性友好、温暖亲和、清晰易用、规整饱满、轻盈灵动**。
- 圆润：图标、卡片、按钮减少锐角，曲率连续。
- 规整：信息层级清楚，对齐稳定，模块不散。
- 饱满：卡片不能空、碎、轻飘，视觉重量要撑住模块。
- 轻盈：背景、分割、投影克制，界面不脏、不厚、不吵。

### 颜色角色（禁止混用）
| 角色 | Fallback | 用途 |
|---|---|---|
| 品牌红 | `#ff4d88` | 主操作、Tab 选中、焦点、核心数据强调 |
| 品牌浅底 | `rgba(255,77,136,0.08)` | 次按钮、轻提示、选中底 |
| 页面背景 | `#f2f2f5` | App/H5 页面底色 |
| 卡片白 | `#ffffff` | 卡片、列表、输入框、弹窗 |
| 主文字 | `#323232` | 标题、正文、关键数值 |
| 次文字 | `#666666` | 副标题、描述、辅助信息 |
| 弱文字 | `#999999` | 占位、时间、弱提示 |
| 分割线 | `rgba(0,0,0,0.08)` | 列表、模块分割 |
| 成功/绿 | `#00cc99` | 健康达标、正向反馈 |
| 警告橙 | `#ff9500` | 建议改善、需注意 |
| 危险红 | `#ff4d4d` | 风险、错误、删除 |
| 链接蓝 | `#4f7cae` | 可跳转链接、医疗信息入口 |
| 会员紫金 | `linear-gradient(135deg,#8d54ff,#f2c66d)` | 会员权益区域 |

**品牌红不用于**：图标类目色、大面积背景、业务氛围色、危险语意以外的错误色。

### 业务色映射
| 业务 | 主氛围 |
|---|---|
| 经期 | 品牌红/柔红 |
| 备孕/医疗 | 蓝/雾蓝 |
| 怀孕/排卵 | 紫/柔紫 |
| 育儿 | 黄/暖黄 |
| 会员 | 紫金渐变 |

### 字阶
| 层级 | 字号 | 字重 |
|---|---|---|
| 页面主标题 | 28 | 500/600 |
| 模块标题 | 21 | 500 |
| 导航标题 | 17 | 500 |
| 正文 | 17 | 400 |
| 列表主文 | 15 | 400/500 |
| 辅助说明 | 13/14 | 400 |
| 标签/弱提示 | 11/12 | 400/500 |
| 数据大字 | 32/36/40 | 600 |

字间距默认 0，数字用 `font-variant-numeric: tabular-nums`。

### 布局规则
- 基础网格：8dp；小元素：4dp。
- 页面左右安全边距：12 或 16。
- 模块间距：12/16/24（一级区块用 24，卡片组内用 12）。
- **外空间 ≥ 内空间**。

### 圆角
| Token | 值 | 用途 |
|---|---|---|
| xs | 4 | 小标签、徽标 |
| sm | 8 | 按钮、输入框、普通卡片 |
| md | 12 | 大卡片、弹窗 |
| pill | 999 | 胶囊标签、搜索框、开关 |

### 组件要点
- **主按钮**：品牌红背景、白字、高 40/44、一区域只有一个主操作。
- **按钮禁用**：背景+文字+事件全部禁用，不能只变浅文字。
- **触控区**：至少 44×44。
- **卡片**：白底、8/12 圆角、12/16 内边距，优先通过间距分层，轻投影或无投影。
- **列表**：单行 44/56，双行 64/72，图标/标题/辅助/箭头基线对齐。
- **弹窗**：一个主题、一个主行动、必须有关闭/取消路径。
- **Toast**：1.5-2s，不遮挡主操作。
- **空/错/加载/禁用**：每个页面必须设计这些状态。
- **导航**：左侧返回、居中标题、右侧操作；每个页面必须有可识别返回路径。

### 渐变规则
- 运营卡片高度高于双行列表时可用渐变。
- 双行列表高度以下禁用复杂渐变，用纯色或浅底。
- 不用漂浮色球、模糊光斑、无关抽象背景。

### 验收清单（输出前必查）
- [ ] 是否有返回/取消/关闭路径
- [ ] 主操作唯一且明确
- [ ] 品牌色、语意色、业务色分工清楚，未混用
- [ ] 字号/字重/行高在规范内
- [ ] 8dp 网格稳定，外空间 ≥ 内空间
- [ ] 圆角统一，图标圆润一致
- [ ] 完整状态（加载/空/错误/禁用）
- [ ] 360px 宽度文本换行正常，不溢出
- [ ] 触控区 ≥ 44×44
- [ ] 图片不拉伸
- [ ] 视觉克制、清透、规整，无堆粉色/堆渐变/堆装饰
